# 비법거래소 웹 서비스

## Commit Convention

---

### Commit type

- Feat: 기능 작업
- Refactor: 리팩토링
- Fix: 버그 수정
- Hotfix: 긴급 내용
- Docs: 문서 관련
- Test: 테스트(테스트 코드 추가, 수정, 삭제, 비즈니스 로직에 변경이 없는 경우)
- Build: 빌드 관련
- Revert: 롤백
- Style: CSS 등 사용자 UI 작업
- Chore: 위에서 언급한 것 이외의 작업

### Rule

- Commit type은 [ ]를 이용하여 작성
- 내용은 간결하게 작성
- 한 작업 할 때마다 commit

### Example

- [Feat] 로그인 API 연동
- [Style] 로그인 화면
